# AGP 9.0 built-in Kotlin migration (Flutter projects)

Sources:

- Flutter app-developer guide: <https://docs.flutter.dev/release/breaking-changes/migrate-to-built-in-kotlin/for-app-developers>
- Flutter index: <https://docs.flutter.dev/release/breaking-changes/migrate-to-built-in-kotlin>
- Android underlying docs: <https://developer.android.com/build/migrate-to-built-in-kotlin>
- AGP 9.0 release notes: <https://developer.android.com/build/releases/agp-9-0-0-release-notes>

## What changed

AGP 9.0 (January 2026) enables built-in Kotlin by default. The separately-applied Kotlin Gradle Plugin (`org.jetbrains.kotlin.android`, aka `kotlin-android`) now conflicts with the built-in support.

Typical errors after upgrading without migrating:

```
Failed to apply plugin 'org.jetbrains.kotlin.android'.
  > Cannot add extension with name 'kotlin', as there is an extension already registered with that name.
```

```
Failed to apply plugin 'com.jetbrains.kotlin.android'
  > The 'org.jetbrains.kotlin.android' plugin is no longer required for Kotlin support since AGP 9.0.
```

## Flutter's temporary shims

Flutter 3.44 keeps existing projects building through two compatibility shims:

1. KGP-on-AGP-9 shim - lets projects that still apply `kotlin-android` keep building.
2. Legacy DSL flag - lets old DSL types (e.g., `android.kotlinOptions {}`) still parse.

Both shims will be removed in a future Flutter release. Migrating in the 3.44 window is safer than waiting.

## Auto-applied opt-out flags

On the next `flutter run` / `flutter build apk` after upgrading to Flutter 3.44, the Flutter CLI writes these to `<app-src>/android/gradle.properties`:

```properties
android.newDsl=false
android.builtInKotlin=false
```

Behavior notes:

- These are temporary opt-out flags - the goal is to flip them OFF (or delete them) once migration is complete.
- Add-to-app projects (where the Android host is a regular Android app, not generated by Flutter) must add these flags manually - the Flutter migrator runs only inside Flutter-managed Android folders.
- If the app never applied `kotlin-android`, only `android.newDsl=false` is needed and there is nothing else to migrate.
- `android.builtInKotlin=false` will not be honored in AGP 10.0 (expected late 2026).

## Step-by-step migration

Goal: remove the opt-out flags by adopting built-in Kotlin properly.

### Step 1 - Module-level build file

File: `android/app/build.gradle.kts` (or `.gradle` for Groovy).

Remove the `kotlin-android` plugin application.

Kotlin DSL before:

```kotlin
plugins {
    id("com.android.application")
    id("kotlin-android")        // remove
}

android {
    kotlinOptions {              // remove this whole block
        jvmTarget = JavaVersion.VERSION_17.toString()
    }
}
```

Kotlin DSL after:

```kotlin
plugins {
    id("com.android.application")
}

android {
    // ...
}

kotlin {
    compilerOptions {
        jvmTarget = org.jetbrains.kotlin.gradle.dsl.JvmTarget.JVM_17
    }
}
```

Groovy DSL before:

```groovy
apply plugin: 'com.android.application'
apply plugin: 'kotlin-android'   // remove

android {
    kotlinOptions {               // remove
        jvmTarget = JavaVersion.VERSION_17.toString()
    }
}
```

Groovy DSL after:

```groovy
apply plugin: 'com.android.application'

android {
    // ...
}

kotlin {
    compilerOptions {
        jvmTarget = org.jetbrains.kotlin.gradle.dsl.JvmTarget.JVM_17
    }
}
```

### Step 2 - Top-level build file

File: `android/build.gradle.kts` (or `.gradle`).

Remove any top-level Kotlin plugin declaration. Examples:

With version catalogs:

```kotlin
// remove
plugins {
    alias(libs.plugins.kotlin.android) apply false
}
```

Without version catalogs:

```kotlin
// remove
plugins {
    id("org.jetbrains.kotlin.android") version "KOTLIN_VERSION" apply false
}
```

Groovy variant:

```groovy
// remove
plugins {
    id 'org.jetbrains.kotlin.android' version 'KOTLIN_VERSION' apply false
}
```

### Step 3 - Version catalog TOML (if used)

File: `android/gradle/libs.versions.toml`.

```toml
[plugins]
# remove this entry:
kotlin-android = { id = "org.jetbrains.kotlin.android", version.ref = "KOTLIN_VERSION" }
```

### Step 4 - Remove the opt-out flags

File: `android/gradle.properties`.

Delete the auto-added flags once migration is complete:

```properties
# remove these
android.newDsl=false
android.builtInKotlin=false
```

### Step 5 - Add-to-app Android host (only if applicable)

For add-to-app projects, the host Android app uses `alias` style:

```kotlin
plugins {
    alias(libs.plugins.android.application)
    alias(libs.plugins.kotlin.android)    // remove
}

android {
    kotlinOptions {                        // remove
        jvmTarget = JavaVersion.VERSION_17.toString()
    }
}
```

After:

```kotlin
plugins {
    alias(libs.plugins.android.application)
}

android {
    // ...
}

kotlin {
    compilerOptions {
        jvmTarget = org.jetbrains.kotlin.gradle.dsl.JvmTarget.JVM_17
    }
}
```

### Step 6 - Validate

```bash
flutter run
flutter build apk
```

Both must succeed without warnings about KGP or the legacy DSL.

## Special cases

### kotlin-kapt

`org.jetbrains.kotlin.kapt` is incompatible with built-in Kotlin.

Preferred path: migrate to KSP. See <https://developer.android.com/build/migrate-to-ksp>.

Temporary workaround: replace with `com.android.legacy-kapt` (same version as AGP). In `libs.versions.toml`:

```toml
[plugins]
android-application = { id = "com.android.application", version.ref = "AGP_VERSION" }
legacy-kapt = { id = "com.android.legacy-kapt", version.ref = "AGP_VERSION" }
# remove:
# kotlin-kapt = { id = "org.jetbrains.kotlin.kapt", version.ref = "KOTLIN_VERSION" }
```

### kotlin.sourceSets {}

`kotlin.sourceSets {}` and `android.sourceSets.java.directories` are no longer supported. Use the new form:

```kotlin
android.sourceSets.named("main") {
    kotlin.directories += "additionalSourceDirectory/kotlin"
}
```

For variant-specific or generated sources, use the Variant API:

```kotlin
androidComponents.onVariants { variant ->
    variant.sources.kotlin!!.addStaticSourceDirectory("additionalSourceDirectory/kotlin")
    variant.sources.kotlin!!.addGeneratedSourceDirectory(TASK_PROVIDER, TASK_OUTPUT)
}
```

### Modules without Kotlin sources

For multi-module Android builds where some modules are pure Java, disable built-in Kotlin per-module to save build time and avoid the stdlib dependency:

```kotlin
android {
    enableKotlin = false
}
```

### KMP modules

Built-in Kotlin replaces `kotlin-android` only. Kotlin Multiplatform library modules still require:

- `org.jetbrains.kotlin.multiplatform`
- `com.android.kotlin.multiplatform.library`

Using `org.jetbrains.kotlin.multiplatform` together with `com.android.library` or `com.android.application` is no longer allowed when built-in Kotlin is enabled.

## Module-by-module incremental migration

For large projects:

1. Set `android.builtInKotlin=false` in `gradle.properties` (global opt-out)
2. Apply `com.android.built-in-kotlin` plugin to the single module being migrated (same version as AGP)
3. Apply migration steps for that module
4. Repeat
5. Once all modules are migrated: delete `android.builtInKotlin=false` and remove all `com.android.built-in-kotlin` plugin declarations

## Verification commands

```bash
flutter run
flutter build apk --debug
flutter build apk --release
# from android/:
./gradlew :app:dependencies | grep kotlin
```

The dependency tree should show Kotlin coming from `kotlin-stdlib` pulled in by AGP, not from a separate `kotlin-android` plugin chain.
